Shorts Media Keys

A lightweight, zero-dependency Chrome Manifest V3 extension that brings full hardware media key support to YouTube Shorts.

View on GitHub Get the Code

The Problem

By default, YouTube Shorts doesn't map your keyboard's "Next Track" and "Previous Track" media keys to scrolling. You're completely forced to stay inside the browser window to navigate.

The Solution

This extension seamlessly hooks into the browser's native Media Session API. Your hardware media keys now control YouTube Shorts even when Chrome is minimized or not focused.

Features

Play/Pause

Natively supported and deeply integrated into the OS.

Next Track

Automatically scrolls down to the next Short.

Previous Track

Automatically scrolls up to the previous Short.

Zero Bloat

No background scripts constantly chewing memory. Only activates exactly on youtube.com/shorts/.

Privacy First

Only requests the absolute minimum base permissions for YouTube. No tracking, no history reading, no phoning home.

How It Works Under the Hood

Normally, YouTube's own player logic resets custom Media Session bindings on Shorts. To defeat this, the extension bypasses YouTube's strict CSP by injecting a native script directly into the MAIN world.

It hijacks navigator.mediaSession.setActionHandler globally for the tab, ensuring that YouTube cannot overwrite or delete the custom next/prev track key bindings, while leaving normal video playlists untouched.

Manual Installation

The extension is fully compatible with Chrome, Edge, Brave, Opera, and all other Chromium-based browsers. Install it directly from source.

  1. 1 Clone or download the GitHub repository.
  2. 2 Open your browser's extensions page (e.g., chrome://extensions/ or edge://extensions/).
  3. 3 Enable Developer mode using the toggle switch.
  4. 4 Click the Load unpacked button.
  5. 5 Select the downloaded extension folder.

Known Limitation

OS media key priority generally goes to the application that most recently started playing media. If you have another app (like Spotify, Apple Music, or Discord) actively playing sound, Windows/macOS will route your physical keys to that app instead. Pause the background music to let Chrome take priority.
